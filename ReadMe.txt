Из-за этого также везде инстанцируются объекты, что очень плохо.
2) Тесты имеют ввид:
Вид наименования:
TestMethod_ExpectedData_WhatMustReturn
Где:
TestMethod     - Тестируемый метод.
ExpectedData   - Ожидаемые данные.
WhatMustReturn - Что должен вернуть.
В каждом тесте по 3 блока, где:
Arrange - инициализация начальных данных.
Act     - действия.
Assert  - проверки ожидаемых результатов.
Добавил 3 unit теста, в реальности нужно больше.
3) Можно что-нибудь придумать, чтобы название каналов задавались в одном месте.
Можно банально ошибиться в одной букве.
